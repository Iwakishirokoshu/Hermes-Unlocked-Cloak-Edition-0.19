# Cloak Proxy Pool

Минимальное персистентное хранилище прокси, чтобы каждый новый Cloak-профиль получал свежий, неиспользованный прокси из общего пула, а агент не путался кому что выдал.

## Хранилище

Один файл: `~/.hermes/cloak/proxies.json` (формат — массив объектов).

```json
[
  {"url": "http://user:pass@1.2.3.4:8080", "assigned_to": "acc-001", "used_at": "2026-06-19T22:30:00Z"},
  {"url": "http://user:pass@5.6.7.8:8080", "assigned_to": null,      "used_at": null}
]
```

## CLI хелпер

`/root/.hermes/skills/cloak-proxy-pool/pool.py` — единственный исполняемый файл скилла. Все операции атомарные (через временный файл + rename), так что параллельные регистрации не перетрут друг друга.

```bash
# Залить новый пул (мерж — дубликаты по url пропускаются)
python3 pool.py load <<EOF
1.2.3.4:8080:user:pass
5.6.7.8:8080:user:pass
http://user:pass@9.10.11.12:3128
socks5://user:pass@13.14.15.16:1080
EOF

# Атомарно выдать следующий неиспользованный (помечает assigned_to)
python3 pool.py next <profile_name>
# → JSON: {"url": "http://...", "index": 1}

# Освободить (если профиль удалён)
python3 pool.py release <profile_name>

# Статус: total / free / used
python3 pool.py status
```

## Поддерживаемые форматы при `load`

| Что прислал юзер | Парсится как |
|---|---|
| `1.2.3.4:8080` | `http://1.2.3.4:8080` |
| `1.2.3.4:8080:user:pass` | `http://user:pass@1.2.3.4:8080` |
| `user:pass@1.2.3.4:8080` | `http://user:pass@1.2.3.4:8080` |
| `http://...` / `https://...` / `socks5://...` | как есть |

Пустые строки и комментарии (`#`) игнорируются. Один прокси на строку.

## Когда юзается

1. **Юзер вставил пул** («вот пул, ротируй» / «загрузи прокси»):
   - Выполнить загрузку через pool.py и подтвердить количество из status.

2. **Юзер попросил создать Cloak-профиль с proxy из общего пула**:
   - Для вызова `cloak_create_profile` или `cloak_set_active` передать только
     `use_pool=true`. Инструмент сам атомарно выберет и закрепит свободный
     proxy за именем профиля.
   - Не вызывай перед этим `pool.py next` или
     `cloak_proxy_pool(action="next")`: получится лишняя резервация.
   - Ручной `next` оставь только для внешнего workflow, который не использует
     Cloak toolset.

3. **Профиль нужно закрыть или остановить после явного решения оператора**:
   - Вызвать `cloak_stop(profile=...)`; инструмент освободит reservation
     proxy из пула.
   - Ручной `pool.py release` нужен только для старого внешнего workflow,
     который сам делал `pool.py next`.

## Связи с другими скиллами

Операции создания и удаления Cloak-профилей должны брать и освобождать прокси через этот общий пул, а не вести отдельную несинхронизированную ротацию.
